# Git とは何か

**Git** はバージョン管理システムの一つです。
最初は難しいかもしれませんが、ゆっくり理解していきましょう。

Git によって可能になることとして、大まかに次の 2 つがあります。

-   バージョン管理
-   共同開発

それぞれどのようなものか、簡単に説明します。

## バージョン管理とは

みなさんは、こんな状況に出くわしたことはないでしょうか。

```txt
ずっと前からレポートを書いていて、今日も途中から書いていたけど、
なんか論点からズレてきているようで……昨日の状態に戻したいな……
いろんなところを編集しすぎてわけが分からなくなっちゃった！
```

基本的にどんなファイルも上書き保存してしまえば元の状態に戻すのは困難です。

これはソースコードを書く際にも起こりえます。

```txt
昨日まではうまく表示できていたけど、今日編集したらエラーを吐くようになってしまった。
```

ということがかなり頻繁に起こります。
どこを編集したかが分かれば、解決がスムーズになります。

そこで、その時その時の状態をセーブポイントのように保存しておき、
**あとで戻れるようしたり、どのような変更が足されたのかが分かるようにしておく** のです。
これが **バージョン管理** です。

## 共同開発

ソースコードを共同で開発するにはどうしたらよいでしょう。
ソースコード全体を zip ファイルか何かで Slack で共有すればよいでしょうか。
これだと今誰が編集しているのかわかりにくいし、同時に編集することは難しいですね。

では、Google ドキュメントのようにリアルタイムで共同編集できるようにしたらどうでしょうか。
一見良さそうに思えますが、たとえばプレビューをするときに他人が JavaScript
などを編集している途中だと、コンパイルできずエラーが出るでしょう。
こうなると他人を待つことになってしまいます。

<Callout>
    ある言語で書かれたコードを別の形式や言語の一連の命令に変換することを
    **コンパイル** (Compile) といいます。 Web
    サイトを開発する際には、実際に皆さんの PC
    に届くコードを直接編集するのではなく、開発者が書きやすい別の形式でコードを書くことがよくあります。
    しかし、そのままではどのような表示になるかがわからないため、開発サーバーを立て、プレビューをするのです。
</Callout>

**自分の編集する箇所だけで完結させながら、なおかつ複数人で同時に開発を進めたい** ですね。
Git なら「ブランチ」の機能を使ってこれを実現できます。
楽しみにしておいてください。

## Git の仕組み

これらを実現する Git はどのような仕組みになっているのでしょう。

全体図を示すと、次の図のようになります。

何が何だかわからないと思うので、少しずつ解説していきます。

### ワーキングツリー

**ワーキングツリー** は Git で管理下にあるディレクトリやファイルのことであり、実際に編集するディレクトリやファイルです。
VS Code で開いたディレクトリ（上の図では「新歓サイト 2025」 ）の中身のうち、
`.git` ディレクトリ以外の部分が該当します。
（Git で管理されていないディレクトリには `.git` はありません。）

ワーキングツリーは「ワークツリー」や「作業ツリー」などと呼ばれることもありますが、
この講習サイトではワーキングツリーと統一します。

### コミット

上の方で、バージョン管理のために開発が進む都度セーブポイントのように記録していくと説明しましたが、
このセーブポイントこそが **コミット** です。

各コミットはそれぞれの 1 つ前のコミットと紐づいており、 **前のコミットからの増分の形で保存されています** 。
コミットは全体として一連の流れとして捉えられるようになります。
上の図ではオレンジ色の玉として表現しています。

図では玉が分岐していますが、これは共同開発をする際に一度別々に開発し、後で統合するという操作をしています。
このように、コミットの流れは分岐することがありますが、あとで統合することができます。
分岐したそれぞれを **ブランチ** と呼びますが、詳しくは別のページで説明します。

また、コミットを作ること自体を「コミットする」と言います。
コミットは名詞としても動詞としても使われて混乱するかもしれませんが、2 つ意味があることを知っておきましょう。

### リポジトリ

**リポジトリ** はコミットの流れやブランチなどを管理しているものです。
リポジトリには 2 種類あり、
自分のディレクトリに入っているものが「ローカルリポジトリ」、
インターネット上にあり他人からもアクセスできるのが「リモートリポジトリ」です。
FairWind のリモートリポジトリは GitHub のサーバーに作っています。

リモートリポジトリとローカルリポジトリは同じ内容が保存されています。
同期させたい場合は手動で同期させる必要があります。
「プル」や「プッシュ」によって行いますが、後ほど詳しく説明します。
