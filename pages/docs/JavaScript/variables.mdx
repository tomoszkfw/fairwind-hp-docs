# 変数

## 変数とは

**変数** を用いると、値を一時的に保存し、再利用することができます。
プログラミングをする上では、変数には文字列等を含め、全ての種類の「値」を格納することができます。

## 変数の宣言と利用

以下のコードを見てみましょう。

```js
let name = "John";
document.write(name);
```

これを HTML ファイルで読み込み、ブラウザで表示させると `John` と表示されるはずです。

`let` は、変数を宣言するためのキーワードです。1 行目では、
`name` という名前の変数を宣言し、そこに文字列 `"John"` を代入しています。
`=` は代入を表し、左側に指定した変数に対し、右側に指定された値を代入します。

2 行目では、変数 `name` が評価され、代入されていた文字列値 `"John"` が画面に表示されます。

import { Callout } from "nextra/components";

### `let` と `const`

変数を宣言するキーワードには、`let` 以外にも `const` があります。 記法自体は
`let` と同様ですが、少し違いがあります。まずは `let`
を用いた場合を見てみましょう。

```js
let price = 500;
price = price * 1.1;
console.log(price);
```

<Callout type="info">
`price = price * 1.1;{:js}` という記述は奇妙に見えるかもしれませんが、
`=` は **代入** を表しているので、これでちゃんと動作します。
</Callout>

次に `const` を用いた場合を見てみます。以下のコードはエラーになるはずです。

```js
const price = 500;
price = price * 1.1;
console.log(price);
```

これは `const` で定義された変数には **値を再代入できない** という性質によるものです。

## 変数の命名

変数に適切な命名をすることは非常に重要です。以下の例を見てみましょう。

```js
const a = 500;
const b = 3;
const c = a * b;
document.write(c);
```

```js
const itemPrice = 500;
const itemCount = 3;
const totalPrice = itemPrice * itemCount;
document.write(totalPrice);
```

前者と後者のコードは、同じ計算を行っています。しかし、前者のコードでは `a` や `b` 、`c` という変数名が何を表しているのかがわかりません。一方、後者のコードでは、`itemPrice` や `itemCount` 、`totalPrice` という変数名から、どのような計算を行っているのかがすぐに分かります。共同で開発を行うときや、後からコードを読み返すときに、変数名が適切に付けられていると、コードの理解が容易になります。

上の例では、item price という 2 語を `itemPrice` のように 2 語目の頭文字を大文字にして繋げて記述しています。

このように、複数の単語にわたるフレーズを、2 語目以降の先頭の文字を大文字にして結合する命名規則を、 **キャメルケース** と呼びます。

JavaScript の変数名には、通常キャメルケースが用いられます。 スネークケース（例： `item_price` ）やパスカルケース（例： `ItemPrice` ）などの命名規則が利用される文法もあります。
