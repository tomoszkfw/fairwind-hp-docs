# CSS による配置

CSS を学ぶ上での一つの目玉として、各 HTML 要素をどのように配置するか、という問題を取り上げます。とても大事だけど、思ったようにいかないことも多いので、色々試しながら進んでいってください。

## CSS ボックスモデル

まず、 **CSS ボックスモデル** という概念を紹介します。ブラウザの開発者ツールを開いてみましょう。

<Callout type="info">
ここで、ブラウザの開発者ツールの開き方をおさらいしておきましょう。開発者ツールを開くには、以下のキーを操作すれば良いのでした。

-   Windows : `Ctrl` + `Shift` + `I` または `F12`
-   macOS : `command` + `option` + `I`

また、画面操作によって開く方法もありますが、ブラウザごとの差が大きいため割愛します。詳しくは使っているブラウザに合わせて、自分で調べてみてください。

</Callout>

### padding

パディングは境界とコンテンツ領域の間に位置し、 **コンテンツを境界から離す** ために使用します。

マージンとは異なり、負のパディングを指定することはできません。パディングの部分には、コンテンツと同じ背景が表示されます。

以下のように設定することができます。

```css
/* 例 1 : 4 辺の padding を等しくする場合 */
.box {
    padding: 20px;
}

/* 例 2 : 上下・左右で padding を違う値にする場合 */
.box {
    /* 上下 | 左右 */
    padding: 10px 20px;
}

/* 例 3 : 4 辺それぞれに padding を設定する場合 */
.box {
    /* 上 | 右 | 下 | 左 */
    padding: 1rem 20px 0 auto;
}
```

<Callout>
padding を設定するには、以下の例のように `padding-top` プロパティなど、 4 辺別々のプロパティを用いることもできます。
    <details>
        <summary>4 辺別々のプロパティを用いる例</summary>

        ```css
        .box {
            padding-top: -40px;
            padding-right: 30px;
            padding-bottom: 40px;
            padding-left: 4em;
        }
        ```

    </details>

</Callout>

### margin

margin とは、ボックスの周りに「見えない空間」を設けるようなものです。 つまり、 **他の要素を遠ざける** 効果があるということです。

margin の値は正でも負でもよく、ある要素に負のマージンを設定すると、ページ上の他の部分と重ねる表示を実現できたりします。

以下のように設定することができます。

```css
/* 例 1 : 4 辺の margin を等しくする場合 */
.box {
    margin: 20px;
}

/* 例 2 : 上下・左右で margin を違う値にする場合 */
.box {
    /* 上下 | 左右 */
    margin: 10px 20px;
}

/* 例 3 : 4 辺それぞれに margin を設定する場合 */
.box {
    /* 上 | 右 | 下 | 左 */
    margin: 1rem 20px 0 auto;
}
```

<Callout>
margin を設定するには、以下の例のように `margin-top` プロパティなど、 4 辺別々のプロパティを用いることもできます。
    <details>
        <summary>4 辺別々のプロパティを用いる例</summary>

        ```css
        .box {
            margin-top: -40px;
            margin-right: 30px;
            margin-bottom: 40px;
            margin-left: 4em;
        }
        ```

    </details>

</Callout>

### border

border (境界) は、ボックスの margin と padding の間に描かれる **境界線** です。以下のように設定することができます。

```css
.border {
    /* 幅 | スタイル | 色 */
    border: 2px solid #0693e3;
}
```

<Callout type="info">
border の設定には、実現したい表示に応じてさまざまなプロパティを用いることができます。

<details>
    <summary>その他の border の設定</summary>

    それぞれの辺のプロパティを個別に設定するには、以下のプロパティを使用します。

    - `border-top`
    - `border-right`
    - `border-bottom`
    - `border-left`

    すべての辺の幅、スタイル、色を設定するには、以下のプロパティを使用します。

    - `border-width`
    - `border-style`
    - `border-color`

    単一の辺の幅、スタイル、色を設定するには、より詳細な以下の個別指定プロパティのいずれかを使用します。

    - `border-top-width`
    - `border-top-style`
    - `border-top-color`
    - `border-right-width`
    - `border-right-style`
    - `border-right-color`
    - `border-bottom-width`
    - `border-bottom-style`
    - `border-bottom-color`
    - `border-left-width`
    - `border-left-style`
    - `border-left-color`

以下に例を示します。

```css filename="CSS"
.container {
    border-top: 5px dotted green;
    border-right: 1px solid black;
    border-bottom: 20px double rgb(23 45 145);
}

.box {
    border: 1px solid #333333;
    border-top-style: dotted;
    border-right-width: 20px;
    border-bottom-color: hotpink;
}
```

```html filename="HTML"
<div class="container">
    <div class="box">Change my borders.</div>
</div>
```

</details>
</Callout>

## レスポンシブデザイン

CSS による配置を学ぶ上での集大成とも言えるのがレスポンシブデザインの話だと思います。新歓サイトの例を見てみましょう。

また、このサイト自体もレスポンシブデザインに対応しているはずです。デバイスを切り替えて確かめてみましょう。
このように、デバイスによって表示を変えることによって、どのデバイスでも快適に見られるサイトを作ることが重要になります。これを実現するのがレスポンシブデザイン、というわけです。

### レスポンシブデザインの例

レスポンシブデザインの作り方の例を見ていきましょう。

1. スマホでも見切れない幅の要素を、配置を変えることによって PC 対応させる

    - [https://welcome.fairwind-ut.com/members/](https://welcome.fairwind-ut.com/members/) を見てみると、一人一人の情報が書かれたブロック（このようなコンポーネントを Card ということがあります）はスマホの画面にも入りきる幅（〜300px が目安です）となっていて、その並べ方を変えていることがわかります。
    - `display: flex{:css}` を指定し、入り切らなくなったら改行する（`flex-wrap: wrap{:css}`）ことによって実現されています。このようにするときには、それぞれの要素の大きさを px（ピクセル）単位で絶対的に指定したとしても問題がないことがわかると思います。

1. CSS grid を活用した場合

    - CSS で `display: grid{:css}` を指定すると、画面を縦横に分割して配置ができるようになります。
    - これを応用すると、たとえば画面を横に 12 分割しておいて、画面幅が小さいときは一つの要素が 12 個の grid を占めるように、それより大きくなったら 6 個、4 個、3 個……と減らしていけば、自然な見え方になる、というわけです。
    - 現在はイベントが終了しているためわかりづらいですが、https://welcome.fairwind-ut.com/events/ で使用されていました。

1. メディアクエリを利用する方法

    - 幅が違うメディア（デバイス）に対して、それぞれどのように表示するかを指定すれば良い、という考え方もできると思います。

```css
@media screen and (max-width: 910px) {
    .desktop {
        display: none;
    }
    .mobile {
        display: block;
    }
}
```

    - このように先頭でどのようなデバイスに対する指定かを書くことができます。この例では、幅 910 ピクセル以下のデバイスに対して、desktop というクラスがついた要素を非表示に、mobile というクラスがついた要素を表示するように、それぞれ指定しています。（新歓サイトのナビゲーションバー/ハンバーガーメニューの切り替えの設定に相当します。）
