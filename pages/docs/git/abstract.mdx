# Git とは何か
**Git** はバージョン管理システムの一つです。
最初は難しいかもしれませんが、ゆっくり理解していきましょう。

Git によって可能になることとして、大まかに次の2つがあります。
-   バージョン管理
-   共同揮発

それぞれどのようなものか、簡単に説明します。

## バージョン管理とは
みなさんは、こんな状況に出くわしたことはないでしょうか。
```txt
ずっと前からレポートを書いていて、今日も途中から書いていたけど、
なんか論点からズレてきているようで... 昨日の状態に戻したいな...
いろんなところを編集しすぎてわけが分からなくなっちゃった！
```
基本的にどんなファイルも上書き保存してしまえば元の状態に戻すのは困難です。

これはソースコードを書く際にも起こりえます。
```txt
昨日まではうまく表示できていたけど、今日編集したらエラーを吐くようになってしまった。
```
ということがかなり頻繁に起こります。
どこを編集したかが分かれば、解決がスムーズになります。

そこで、その時その時の状態をセーブポイントのように保存しておき、
**あとで戻れるようしたり、どのような変更が足されたのかが分かるようにしておく**のです。
これが**バージョン管理**です。

## 共同開発
ソースコードを共同で開発するにはどうしたらよいでしょう。
ソースコード全体を zip ファイルか何かで Slack で共有すればよいでしょうか。
これだと今誰が編集しているのかわかりにくいし、同時に編集することは難しいですね。

では、Google ドキュメントのようにリアルタイムで共同編集できるようにしたらどうでしょうか。
一見良さそうに思えますが、例えばプレビューをする際他人が JavaScript などを編集している途中だと、コンパイルできずエラーが出るでしょう。
こうなると他人を待つことになってしまいます。

**自分の編集する箇所だけで完結させながら、なおかつ複数人で同時に開発を進めたい**ですね。
Git なら「ブランチ」の機能を使ってこれを実現できます。
楽しみにしておいてください。
